# DevMind.ShellEncodingProbe

One-off diagnostic to test whether setting `StandardOutputEncoding` / `StandardErrorEncoding` on the `ProcessStartInfo` resolves a bug where DevMind's `ShellRunner` captures empty output from native Windows executables (e.g. `git`) when invoked via `powershell.exe` with `RedirectStandardOutput = true`. The probe builds two `ProcessStartInfo` instances with the exact same flag set used by `DevMind.Core.ShellRunner.ExecuteAsync` (lines 85–93 of `ShellRunner.cs` v1.4), runs `powershell.exe -NoProfile -NonInteractive -Command "git --version"` twice — once with the encoding properties left at their default, once with them set to `Encoding.UTF8` — and reports the captured byte counts and content for each. **This project is not part of the production build**: it is excluded from `DevMind.slnx`, has no references outside the BCL, and is intended to be run once and either deleted or kept alongside the diagnostic note that triggered it.
